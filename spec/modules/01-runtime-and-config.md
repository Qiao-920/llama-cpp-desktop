# 模块规格：运行时与配置

## 模块目标

把 llama.cpp 的启动参数、模型路径、端口、模板参数、性能参数收进桌面端设置中心，并保证默认值不会轻易把用户电脑打满。

## 用户故事

1. 用户选择 `llama-server.exe` 所在目录和 GGUF 模型后，可以点击保存并启动。
2. 用户可以看到 OpenAI Base URL 和 Chat Completions URL。
3. 用户不知道命令行参数时，仍然能用安全默认值启动。
4. 用户填高风险参数时，界面必须给出风险提示。

## 配置职责

| 配置 | 责任 |
| --- | --- |
| llama.cpp 原文件目录 | 找到 `llama-server.exe` 和相关 DLL。 |
| 模型文件 | 作为 `--model` 或等价参数传给 llama-server。 |
| mmproj 文件 | 视觉模型才需要；没有时不应假装支持图片理解。 |
| host/port | 决定本地服务监听地址和 OpenAI 兼容 URL。 |
| ctx_size | 控制上下文大小，直接影响内存。 |
| n_predict | 控制最大输出，`-1` 表示按模型/服务默认策略。 |
| gpu_layers | 控制 GPU 分层，错误配置可能导致启动失败。 |
| chat_template_kwargs | 控制模板行为，例如 Qwen 思考模式。 |
| extra_args | 高级用户补充参数，但必须避免与核心参数冲突。 |

## 默认值要求

| 项 | 默认值 | 原因 |
| --- | --- | --- |
| host | `127.0.0.1` 或用户明确选择的监听地址 | 避免默认暴露到局域网。 |
| port | `8080` | 与项目文档和界面保持一致。 |
| ctx_size | `32768` | 降低内存爆掉概率。 |
| request_timeout_ms | `600000` | 长回答和低速模型不应轻易超时。 |
| n_predict | `-1` | 尊重 llama.cpp 默认输出策略。 |

## 高风险提示

必须提示的情况：

1. `ctx_size > 65536`：提示可能显著增加内存占用。
2. host 为 `0.0.0.0`：提示服务可能被局域网访问。
3. 未设置 mmproj 却上传图片：提示只能作为附件预览，不能保证视觉理解。
4. llama.cpp 目录不存在 `llama-server.exe`：保存可允许，但启动必须报错。

## 验收重点

1. 设置页保存后，重启应用仍能恢复配置。
2. 启动服务后状态栏显示的 URL 与设置一致。
3. `ctx_size` 默认值和高风险提示符合 [acceptance.md](../acceptance.md) 的 B 组。
4. 启动失败时错误可读，不把完整内部堆栈塞进聊天记录。
