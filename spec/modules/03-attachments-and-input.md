# 模块规格：附件与输入框

## 模块目标

附件入口要像现代聊天工具一样轻量、可点击、不遮挡；上传后的文件必须以适合类型的方式展示，不能用巨大卡片挤压聊天区。

## 附件入口

点击输入框左下角加号后，菜单应包含：

1. 图片。
2. 音频文件。
3. 文本文件。
4. PDF 文件。
5. 系统消息。
6. MCP 服务器。

## 菜单布局规则

1. 菜单使用浮层/portal 方式渲染，不能被输入框容器裁剪。
2. 菜单优先向上展开；空间不足时允许改变方向，但所有菜单项必须可见。
3. 菜单打开时，加号按钮仍在原位，不推动输入框高度。
4. 禁用功能必须显示禁用态和简短原因。

## 附件展示规则

| 类型 | 展示 |
| --- | --- |
| 图片 | 轻量图片气泡，图片自然尺寸受限，不放进大面积空白卡片。 |
| 文本 | 文件 pill 或紧凑文件卡，显示文件名、类型、大小。 |
| PDF | 文件 pill 或紧凑文件卡，显示文件名、类型、大小。 |
| 音频 | 文件 pill 或紧凑文件卡，并提示需要音频模型或暂不支持。 |
| 系统消息 | 以文本配置方式进入 system，不显示为普通用户聊天附件。 |

## 图片尺寸硬标准

1. 图片预览宽度不超过 `280px` 或 `52vw`，取较小值。
2. 图片预览高度不超过 `320px`。
3. 图片气泡外层不能超过聊天内容宽度的 `70%`。
4. 图片不能把用户文本挤到不可读位置。
5. 文件名超过容器宽度时必须省略号截断。

## 输入框规则

1. 输入框固定在聊天底部附近，但不能盖住最后一条消息。
2. 发送按钮、模型 pill、加号按钮在窄窗口下不能重叠。
3. Shift+Enter 换行，Enter 发送。
4. 生成中发送按钮可变为停止/等待态，但必须有明确视觉反馈。
