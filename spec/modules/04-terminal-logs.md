# 模块规格：终端日志

## 模块目标

终端日志页要像正常终端一样帮助用户判断服务是否启动、模型是否加载、错误在哪里，而不是展示所有流式响应和 prompt 回显。

## 展示风格

1. 黑底。
2. 等宽字体。
3. 日志行保留时间、来源和内容。
4. 纵向可滚动，必要时横向可滚动。
5. 不使用大白色表格展示长日志。

## 默认过滤规则

默认隐藏：

1. `http: streamed chunk: data:` 的大段 JSON。
2. 完整 prompt 回显。
3. 完整 HTML/CSS/JS 代码回显。
4. `que start_loop: waiting for new tasks` 等高频空闲轮询。
5. 重复的 `all slots are idle`。
6. 已经在聊天 UI 展示的完整 assistant 内容。

默认保留：

1. 模型加载信息。
2. CUDA/CPU/内存关键行。
3. 服务监听地址。
4. 端口占用。
5. 模型路径错误。
6. 请求开始、请求完成、生成速度摘要。
7. llama-server 退出和崩溃信息。

## 数量限制

1. 默认可见日志不超过 `520` 行。
2. 如果隐藏或截断了日志，界面必须显示数量，例如“已隐藏/截断 780 条噪音日志”。
3. 未来可以增加“显示全部/复制过滤后日志/复制原始日志”，但默认仍应是过滤视图。
